# ТЗ для фронта: POST /api/v1/cart

## Назначение

Добавление продукта в корзину пользователя с указанием количества в граммах. Эндпоинт требует авторизации.

---

## Запрос

| Параметр | Значение |
|----------|----------|
| **Метод** | `POST` |
| **URL** | `/api/v1/cart` |
| **Content-Type** | `application/json` |
| **Авторизация** | Обязательна (токен/сессия в заголовках, как принято в проекте) |

### Тело запроса (JSON)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `product` | object | да | Объект продукта (из поиска или по штрихкоду) |
| `grams` | number | да | Количество в граммах; должно быть > 0 |

#### Структура объекта `product` (FoodItem)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `id` | string | да | Уникальный ID продукта (из API источника) |
| `name` | string | да | Название продукта |
| `brand` | string | нет | Бренд |
| `category` | string | нет | Категория |
| `description` | string | нет | Описание |
| `nutrition` | object | да | Пищевая ценность на 100 г |
| `serving_size` | string | нет | Размер порции |
| `image_url` | string | нет | URL изображения |
| `source` | string | да | Источник данных: `usda`, `openfoodfacts`, `edamam` |

#### Объект `nutrition` (на 100 г)

| Поле | Тип | Описание |
|------|-----|----------|
| `calories` | number | Ккал |
| `protein` | number | Белки, г |
| `fat` | number | Жиры, г |
| `carbohydrates` | number | Углеводы, г |
| `fiber` | number | Клетчатка, г (опционально) |
| `sugar` | number | Сахар, г (опционально) |
| `sodium` | number | Натрий, мг (опционально) |

### Пример тела запроса

```json
{
  "product": {
    "id": "12345",
    "name": "Молоко 3.2%",
    "brand": "Простоквашино",
    "source": "openfoodfacts",
    "nutrition": {
      "calories": 60,
      "protein": 2.8,
      "fat": 3.2,
      "carbohydrates": 4.7
    },
    "image_url": "https://..."
  },
  "grams": 200
}
```

---

## Успешный ответ

| Параметр | Значение |
|----------|----------|
| **Код** | `201 Created` |
| **Content-Type** | `application/json` |

### Тело ответа

Возвращается одна созданная позиция корзины — объект **CartItem**:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | ID позиции в корзине |
| `user_id` | number | ID пользователя |
| `food_id` | string | Внешний ID продукта |
| `name` | string | Название продукта |
| `brand` | string | Бренд |
| `source` | string | Источник данных |
| `grams` | number | Количество в граммах |
| `nutrition` | object | Пищевая ценность на 100 г (как в `product`) |
| `image_url` | string | URL изображения |
| `created_at` | string (ISO 8601) | Время добавления |
| `total_calories` | number | Калории на указанное кол-во грамм |
| `total_protein` | number | Белки на указанное кол-во грамм |
| `total_fat` | number | Жиры на указанное кол-во грамм |
| `total_carbohydrates` | number | Углеводы на указанное кол-во грамм |

### Пример успешного ответа

```json
{
  "id": 42,
  "user_id": 1,
  "food_id": "12345",
  "name": "Молоко 3.2%",
  "brand": "Простоквашино",
  "source": "openfoodfacts",
  "grams": 200,
  "nutrition": {
    "calories": 60,
    "protein": 2.8,
    "fat": 3.2,
    "carbohydrates": 4.7
  },
  "image_url": "https://...",
  "created_at": "2025-03-11T12:00:00Z",
  "total_calories": 120,
  "total_protein": 5.6,
  "total_fat": 6.4,
  "total_carbohydrates": 9.4
}
```

## Ошибки

Во всех случаях ошибки приходят в виде JSON с полями: `error`, `status`, `message`.

### 401 Unauthorized

Пользователь не авторизован.

```json
{
  "error": "Unauthorized",
  "status": 401,
  "message": "unauthorized"
}
```

**Действие на фронте:** перенаправить на логин / показать форму входа.

---

### 400 Bad Request

Некорректное тело запроса или `grams <= 0`.

Возможные `message`:

- `"invalid body"` — невалидный JSON или не совпадает схема (например, нет `product` или `grams`).
- `"grams must be positive"` — `grams` не передан, не число или ≤ 0.

```json
{
  "error": "Bad Request",
  "status": 400,
  "message": "grams must be positive"
}
```

**Действие на фронте:** подсветить поле «граммы», показать сообщение (например: «Укажите количество больше 0»).

---

### 405 Method Not Allowed

Использован не POST.

**Действие на фронте:** вызывать только метод POST.

---

### 500 Internal Server Error

Ошибка на сервере при добавлении в корзину.

```json
{
  "error": "Internal Server Error",
  "status": 500,
  "message": "failed to add to cart"
}
```

**Действие на фронте:** показать сообщение об ошибке, предложить повторить попытку.

## Рекомендации для фронта

1. **Откуда брать `product`**  
   Использовать тот же объект, что приходит из поиска продуктов (GET поиск) или из запроса по штрихкоду — без преобразований, целиком в `product`.

2. **Валидация до запроса**  
   - `product` — объект с полями `id`, `name`, `nutrition`, `source`.  
   - `grams` — число строго больше 0 (и, при необходимости, разумный верхний предел, например 10000).

3. **После 201**  
   Можно обновить локальное состояние корзины добавленным элементом из ответа или перезапросить список корзины (GET /api/v1/cart).

4. **Заголовки**  
   Передавать заголовок авторизации так же, как для остальных защищённых эндпоинтов (например, `Authorization: Bearer <token>` или cookie — по принятой в проекте схеме).
